易忘点

  • 每个类都有自己的构造方法，若是没有定义，系统会默认提供一个无参的构造方法，若是在类中自己定义了构造方法，最好将默认的构造方法也写出来，因为自定义的构造方法会覆盖默认的
  • 默认的构造方法只能帮你创建对象，无法做你想做的事情
  • 类的四个成员：属性、方法、构造方法、程序块
  • 认为构造方法早于一般方法
  • 一般方法内无法调用构造方法

类的四个成员

  • 属性–静态描述特征（存值）
  • 方法–动态描述行为（做事情）
  • 构造方法–创建当前类对象（做事情，唯一的事情）
  • 程序快 –一个特殊的方法（没名）

this关键字

  • 是一个关键字（指代词）代替某一个对象（当前调用属性或方法时的那个对象），可以调用属性与方法，可以放置在当前类中的任何成员位置
  • 在一个构造方法内可以调用另一个构造方法，通过this（）；的方式调用，省略了构造方法的名字（因为构造方法的名字与类名一致）。必须在程序的第一行。

程序快

  • 作用：与普通方法一样，做事情的
  • 写法：可以认为它是一个没有修饰符、没有返回值、没有名字的特殊方法
  • 用法：块也是需要调用才可以执行的，我们自己调用不到（没有名字） 每一次调用构造方法前，系统会帮我们自动调用一次
